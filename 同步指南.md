# 项目同步指南

## 📋 同步方案选择

根据你的需求，可以选择以下同步方案：

### 方案1：Git版本控制（推荐）⭐

**优点**：
- 版本历史记录完整
- 支持多设备同步
- 可以回退到任意版本
- 支持分支管理
- 免费（GitHub/Gitee/GitLab）

**适用场景**：
- 需要版本控制
- 团队协作
- 代码和文档管理

### 方案2：云盘同步（简单快速）

**优点**：
- 操作简单
- 自动同步
- 无需学习Git

**适用场景**：
- 个人使用
- 快速同步
- 不需要版本控制

### 方案3：混合方案

**优点**：
- Git管理代码和文档
- 云盘备份大文件（PDF、Word等）

**适用场景**：
- 需要版本控制
- 有大文件需要同步

## 🚀 方案1：Git版本控制（推荐）

### 步骤1：初始化Git仓库

在项目根目录执行：

```bash
cd "p:\Cursor Project"
git init
```

### 步骤2：配置Git用户信息（首次使用）

```bash
git config user.name "你的姓名"
git config user.email "你的邮箱"
```

### 步骤3：添加文件到Git

```bash
git add .
git commit -m "初始提交：创建学习系统和工具"
```

### 步骤4：创建远程仓库

**选项A：GitHub（国际）**
1. 访问 https://github.com
2. 创建新仓库（New Repository）
3. 复制仓库地址（如：https://github.com/username/cursor-project.git）

**选项B：Gitee（国内，速度快）**
1. 访问 https://gitee.com
2. 创建新仓库
3. 复制仓库地址（如：https://gitee.com/username/cursor-project.git）

### 步骤5：连接远程仓库并推送

```bash
git remote add origin https://github.com/username/cursor-project.git
git branch -M main
git push -u origin main
```

### 步骤6：在另一台电脑上克隆

```bash
git clone https://github.com/username/cursor-project.git
cd cursor-project
```

### 日常同步操作

**在电脑A上修改后：**
```bash
git add .
git commit -m "更新条款对应矩阵"
git push
```

**在电脑B上获取更新：**
```bash
git pull
```

## ☁️ 方案2：云盘同步（简单快速）

### 使用OneDrive/百度网盘/阿里云盘等

1. **将项目文件夹添加到云盘同步目录**
   - OneDrive：`C:\Users\你的用户名\OneDrive\Cursor Project`
   - 百度网盘：设置同步文件夹
   - 阿里云盘：设置同步文件夹

2. **在另一台电脑上安装相同云盘客户端**
   - 登录相同账号
   - 文件会自动同步

### 注意事项

- ⚠️ 大文件（PDF、Word）可能同步较慢
- ⚠️ 建议定期备份重要文件
- ⚠️ 避免同时在两台电脑上编辑同一文件

## 📝 快速开始脚本

### 初始化Git仓库

创建 `初始化Git仓库.bat`：

```batch
@echo off
chcp 65001 >nul
cd /d "%~dp0"

echo ========================================
echo 初始化Git仓库
echo ========================================
echo.

if exist .git (
    echo [提示] Git仓库已存在
    echo.
) else (
    echo 正在初始化Git仓库...
    git init
    echo [成功] Git仓库已初始化
    echo.
)

echo 请配置Git用户信息（如果还未配置）:
echo git config user.name "你的姓名"
echo git config user.email "你的邮箱"
echo.
echo 然后添加文件并提交:
echo git add .
echo git commit -m "初始提交"
echo.
echo 创建远程仓库后，连接并推送:
echo git remote add origin <仓库地址>
echo git push -u origin main
echo.

pause
```

### 同步到远程仓库

创建 `同步到远程.bat`：

```batch
@echo off
chcp 65001 >nul
cd /d "%~dp0"

echo ========================================
echo 同步到远程仓库
echo ========================================
echo.

git add .
git commit -m "更新项目文件"
git push

echo.
echo [完成] 已同步到远程仓库
pause
```

### 从远程仓库拉取

创建 `从远程拉取.bat`：

```batch
@echo off
chcp 65001 >nul
cd /d "%~dp0"

echo ========================================
echo 从远程仓库拉取更新
echo ========================================
echo.

git pull

echo.
echo [完成] 已从远程仓库拉取最新更新
pause
```

## 🆘 常见问题

### Q1: 两台电脑同时编辑了同一文件怎么办？

**A**: Git会检测到冲突，需要手动解决：
```bash
git pull  # 拉取时如果有冲突
# 编辑冲突文件，解决冲突标记
git add .
git commit -m "解决冲突"
git push
```

### Q2: 大文件（PDF、Word）同步很慢怎么办？

**A**: 
1. 使用 `.gitignore` 忽略大文件
2. 使用云盘同步大文件
3. 或使用Git LFS（Large File Storage）

### Q3: 忘记提交就切换电脑了？

**A**: 
1. 回到原电脑提交并推送
2. 或使用U盘复制未提交的文件
3. 或使用云盘临时同步

### Q4: 如何查看修改历史？

**A**: 
```bash
git log          # 查看提交历史
git log --oneline  # 简洁版本
git diff         # 查看未提交的修改
```

## 💡 最佳实践建议

1. **定期提交**：每天工作结束后提交一次
2. **提交信息清晰**：使用有意义的提交信息
3. **先拉取再推送**：在推送前先拉取，避免冲突
4. **大文件单独处理**：PDF、Word等大文件用云盘同步
5. **备份重要文件**：定期备份到多个位置
