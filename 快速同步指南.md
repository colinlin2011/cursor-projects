# 快速同步指南（简化版）

## 🚀 三步快速同步

### 第一步：在当前电脑上初始化Git

1. **双击运行**：`初始化Git仓库.bat`
2. **配置用户信息**（如果还未配置）：
   ```bash
   git config user.name "你的姓名"
   git config user.email "你的邮箱"
   ```
3. **创建远程仓库**：
   - 访问 https://github.com 或 https://gitee.com
   - 创建新仓库
   - 复制仓库地址

4. **连接并推送**：
   ```bash
   git add .
   git commit -m "初始提交"
   git remote add origin <你的仓库地址>
   git branch -M main
   git push -u origin main
   ```

### 第二步：在另一台电脑上克隆

```bash
git clone <你的仓库地址>
cd cursor-project
```

### 第三步：日常同步

**在电脑A上修改后：**
- 双击运行：`同步到远程.bat`

**在电脑B上获取更新：**
- 双击运行：`从远程拉取.bat`

---

## 📁 大文件同步（PDF、Word、Excel）

由于PDF、Word、Excel文件较大，Git同步较慢，建议使用云盘同步：

### 需要云盘同步的文件

```
materials/01_Standards/*.pdf
materials/01_Standards/*.docx
materials/04_工作文档/*.xlsx
```

### 云盘同步方法

1. **OneDrive**（Windows自带）
   - 将 `materials` 文件夹添加到OneDrive
   - 在另一台电脑上登录相同账号

2. **百度网盘/阿里云盘**
   - 上传 `materials` 文件夹
   - 在另一台电脑上下载

3. **U盘/移动硬盘**
   - 直接复制 `materials` 文件夹

---

## ⚡ 最快同步方案

### 方案A：Git + 云盘（推荐）

- **Git同步**：代码、Markdown文档（快速）
- **云盘同步**：PDF、Word、Excel（大文件）

### 方案B：纯云盘（最简单）

- 将整个项目文件夹放在云盘同步目录
- 自动同步所有文件

## 🔧 如果遇到问题

### Git相关

1. **未安装Git**：
   - 下载：https://git-scm.com/download/win
   - 或使用：`winget install Git.Git`

2. **连接远程仓库失败**：
   - 检查网络连接
   - 检查仓库地址是否正确
   - 检查是否已登录（GitHub需要Token）

### 云盘相关

1. **同步速度慢**：
   - 这是正常的，大文件需要时间
   - 可以只同步修改的文件

2. **文件冲突**：
   - 避免同时在两台电脑上编辑同一文件
   - 编辑前先检查是否有更新

## 💡 推荐工作流程

### 每天开始工作前

1. 运行 `从远程拉取.bat` 获取最新更新
2. 检查云盘是否有新的大文件

### 每天结束工作后

1. 运行 `同步到远程.bat` 提交代码和文档
2. 确保云盘已同步大文件
