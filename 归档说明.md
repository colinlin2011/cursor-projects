# 项目归档说明

## 📦 归档方案

为了便于管理多个 Python 工具项目，建议采用以下目录结构：

```
Cursor Project/
├── tools/                    # 所有工具项目的目录
│   ├── excel-processor/      # Excel 处理工具（当前项目）
│   │   ├── main.py
│   │   ├── launcher.py
│   │   ├── config.py
│   │   ├── utils/
│   │   ├── requirements.txt
│   │   ├── README.md
│   │   └── ...
│   ├── tool-2/              # 第二个工具（待创建）
│   └── tool-3/              # 第三个工具（待创建）
├── README.md                # 总项目说明
└── .gitignore               # 全局忽略文件
```

## 🚀 归档步骤

### 方法一：使用归档脚本（推荐）

1. 运行 `归档项目.bat` 脚本
2. 脚本会自动：
   - 创建 `tools/excel-processor/` 目录
   - 移动所有项目文件到新目录
   - 保留打包后的 exe 文件（可选）

### 方法二：手动归档

1. 创建目录结构：
   ```bash
   mkdir tools
   mkdir tools\excel-processor
   ```

2. 移动文件（保留 exe 在 dist 目录）：
   ```bash
   move main.py tools\excel-processor\
   move launcher.py tools\excel-processor\
   move config.py tools\excel-processor\
   move utils tools\excel-processor\
   move requirements.txt tools\excel-processor\
   move build_exe.* tools\excel-processor\
   move *.md tools\excel-processor\
   ```

3. 清理临时文件（可选）：
   ```bash
   rmdir /s /q build
   rmdir /s /q dist
   rmdir /s /q __pycache__
   ```

## 📝 归档后的使用

归档后，每个工具都是独立的项目：

1. **开发模式**：
   ```bash
   cd tools\excel-processor
   streamlit run main.py
   ```

2. **打包**：
   ```bash
   cd tools\excel-processor
   build_exe.bat
   ```

3. **运行 exe**：
   - exe 文件位于 `tools\excel-processor\dist\Excel处理工具.exe`

## 🎯 优势

- ✅ 项目结构清晰，每个工具独立
- ✅ 便于版本管理和备份
- ✅ 可以同时开发多个工具
- ✅ 避免文件混乱
- ✅ 便于分享和分发单个工具
