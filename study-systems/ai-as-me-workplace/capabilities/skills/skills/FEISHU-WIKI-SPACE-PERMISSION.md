# 飞书Wiki知识库权限配置指南

## 问题说明

错误码`131006`表示应用对知识库或节点没有编辑权限。即使应用权限（`wiki:wiki`、`wiki:node:create`）已经授权，还需要在知识库中给应用授权编辑权限。

## 错误信息

```
错误码：131006
错误信息：permission denied: wiki space permission denied, tenant needs edit permission
```

## 解决方法

### 方法1：在知识库中授权应用（推荐）

#### 步骤1：打开知识库设置

1. 在飞书中打开目标知识库
2. 点击知识库右上角的**设置**图标（齿轮图标）或**更多**菜单（三个点）
3. 选择**权限设置**或**成员与权限**

#### 步骤2：添加应用为成员

1. 在权限设置页面，找到**成员**或**协作者**列表
2. 点击**添加成员**或**邀请成员**
3. 在搜索框中输入应用名称或应用ID：`cli_a9c92ca516f99bd9`
4. 选择应用，并设置权限为**编辑者**或**管理员**
5. 确认添加

#### 步骤3：验证权限

添加完成后，重新运行创建文档的脚本。

### 方法2：在目录中授权应用

如果只需要在特定目录创建文档：

1. 在飞书Wiki中，进入目标目录
2. 点击目录右上角的**更多**菜单（三个点）
3. 选择**权限设置**或**分享设置**
4. 添加应用为**编辑者**或**管理员**
5. 确认保存

### 方法3：通过知识库分享链接授权

1. 在知识库设置中，找到**分享**或**链接分享**
2. 生成分享链接，并设置权限为**可编辑**
3. 应用可以通过分享链接访问知识库

## 权限级别说明

- **查看者**：只能查看，不能创建或编辑
- **编辑者**：可以创建、编辑文档和节点
- **管理员**：拥有所有权限，包括管理成员和设置

**建议**：至少设置为**编辑者**，才能创建文档。

## 验证步骤

1. 确认应用已在知识库成员列表中
2. 确认应用权限为**编辑者**或**管理员**
3. 重新运行创建文档脚本

## 常见问题

### Q1: 找不到应用？

**A**: 
- 确保应用已发布或已添加到企业应用列表
- 尝试使用应用ID搜索：`cli_a9c92ca516f99bd9`
- 如果找不到，可能需要先在飞书开放平台将应用添加到企业

### Q2: 添加应用后仍然没有权限？

**A**: 
- 检查应用权限级别是否为**编辑者**或**管理员**
- 等待几分钟让权限生效
- 重新获取token（权限变更后可能需要刷新token）

### Q3: 如何批量授权多个知识库？

**A**: 
- 可以在飞书开放平台的应用管理页面，查看应用可以访问的知识库列表
- 或者使用API查询知识库列表，然后逐个授权
