## 抽象能力

### 简单化能力

- **能力描述**：从复杂问题中提取本质，去除冗余和表面细节的能力
- **使用频率**：高
- **典型案例**：
  - [DEC-001](work/decisions/decisions/20260113-pandoc-docker-decision.md)：将"Docker方案太复杂"简化为"需求-方案匹配问题"
- **改进方向**：
  - 提高核心要素识别的准确性
  - 增强去伪存真的能力
  - 建立更多简化模型

### 根节点问题识别能力

- **能力描述**：识别问题根本原因，而非表面症状的能力
- **使用频率**：中
- **典型案例**：
  - [DEC-001](work/decisions/decisions/20260113-pandoc-docker-decision.md)：识别出"投入产出比不匹配"这一根节点问题
- **改进方向**：
  - 提高问题树构建的准确性
  - 增强因果链追溯能力
  - 建立根节点问题库

### 泛化能力

- **能力描述**：将具体问题抽象为通用模式，形成可复用知识的能力
- **使用频率**：中
- **典型案例**：
  - [DEC-001](work/decisions/decisions/20260113-pandoc-docker-decision.md)：提取出"实用性优先决策模式"
- **改进方向**：
  - 提高模式识别的准确性
  - 增强模式抽象能力
  - 建立泛化模式库

### 表示空间定位能力

- **能力描述**：在知识空间中定位和连接问题，建立知识网络的能力
- **使用频率**：低（新能力）
- **典型案例**：
  - [DEC-001](work/decisions/decisions/20260113-pandoc-docker-decision.md)：将问题定位到"技术方案选择"领域
- **改进方向**：
  - 建立完整的知识领域划分
  - 增强知识连接识别能力
  - 建立知识图谱可视化

## 抽象方法库

### 简单化方法

1. **核心要素提取**：识别问题的核心要素
2. **抽象层次提升**：从具体问题抽象到一般问题
3. **模型化**：将问题转化为简化模型

### 根节点问题识别方法

1. **5Why分析法**：连续问5个"为什么"
2. **问题树构建**：从表面问题向下追溯
3. **因果链分析**：分析问题的因果链
4. **系统思考**：从系统角度思考问题

### 泛化方法

1. **模式识别**：识别问题中的重复模式
2. **抽象建模**：建立抽象模型
3. **场景归纳**：归纳相似场景

### 表示空间定位方法

1. **核心概念识别**：从问题中提取核心概念
2. **知识领域确定**：确定问题所属的知识领域
3. **知识连接建立**：找到与已有知识的连接点
4. **知识网络更新**：在知识网络中记录新连接
