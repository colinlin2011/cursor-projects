## 简化模型说明

简化模型是通过"简单化（本质抽象）"步骤提取的问题本质模型。这些模型去除了问题的冗余和表面细节，保留了核心要素，帮助我们快速理解问题的本质。

## 简化模型详情

### SIMPLE-001：需求-方案匹配模型

- **模型描述**：将技术方案选择问题简化为"需求复杂度 vs 方案复杂度"的匹配问题
- **核心要素**：
  - **需求复杂度**：需求的复杂程度（简单/中等/复杂）
  - **方案复杂度**：方案的复杂程度（简单/中等/复杂）
  - **匹配度**：需求与方案的匹配程度（高/中/低）
- **本质问题**：当方案复杂度远高于需求复杂度时，会出现投入产出比不匹配的问题
- **适用场景**：
  - 技术方案选择
  - 工具评估
  - 资源投入决策
- **判断标准**：
  - 高匹配度：方案复杂度 ≈ 需求复杂度
  - 中匹配度：方案复杂度略高于需求复杂度（可接受）
  - 低匹配度：方案复杂度 >> 需求复杂度（不匹配）
- **相关根节点问题**：[ROOT-001](../root-problems/README.md#root-001投入产出比不匹配)
- **相关模式**：[实用性优先模式](../../insights/pattern-discovery.md#模式2实用性优先模式)
- **典型案例**：[DEC-001](../../work/decisions/decisions/20260113-pandoc-docker-decision.md)

## 简化方法

### 方法1：核心要素提取
- 识别问题的核心要素
- 去除非核心要素
- 保留本质特征

### 方法2：抽象层次提升
- 从具体问题抽象到一般问题
- 从现象抽象到本质
- 从细节抽象到规律

### 方法3：模型化
- 将问题转化为数学模型
- 识别关键变量和关系
- 建立简化模型

## 模型应用

### 应用步骤
1. **识别问题类型**：确定问题属于哪个简化模型
2. **提取核心要素**：从问题中提取模型的核心要素
3. **评估匹配度**：使用模型判断匹配度
4. **做出决策**：基于模型评估结果做出决策

### 应用示例

**问题**：Docker方案太复杂，但我想用Pandoc转换PDF

**应用模型**：需求-方案匹配模型

**分析**：
- 需求复杂度：简单（只需要Markdown转PDF）
- 方案复杂度：复杂（需要Docker、镜像、配置等）
- 匹配度：低（方案复杂度 >> 需求复杂度）

**结论**：方案与需求不匹配，应选择更简单的方案
